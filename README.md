Assignment: Smart Transport Management System

Problem Statement
 
A city transport authority wants to develop a Smart Transport Management System to manage vehicles, passengers, bookings, and transport operations.
 
The system should support multiple types of transport vehicles such as:
------------------------------------------------------------------------- 
Bus

Taxi

Metro
 
Each vehicle contains information such as:
------------------------------------------------
Vehicle ID

Vehicle Number

Driver Name

Capacity
--------------------------------------------------
Each passenger contains:
 
Passenger ID

Name

Age

Destination

Functional Requirements
------------------------------------------------

Vehicle Management
 
The system should allow:
 
Adding new vehicles.

Displaying details of all vehicles.

Searching for a vehicle using its ID.

Calculating trip fare based on distance travelled.

Displaying the total revenue generated by a vehicle.
-------------------------------------------------------
Passenger Management
 
The system should allow:
 
Registering passengers.

Storing multiple passenger records.

Searching passengers by Passenger ID.

Displaying all registered passengers.
-----------------------------------------------------------
Booking Management
 
The system should allow:
 
Booking a passenger into a vehicle.

Displaying all passengers assigned to a particular vehicle.

Preventing bookings when vehicle capacity is reached.

Displaying the number of available seats in a vehicle.

Tracking and Maintenance Operations
 
----------------------------------------------------------------------------------------------------------
Implement the system in such a way that these features can be enabled only for applicable vehicle types.
 
Business Rules

A Bus can accommodate a maximum of 50 passengers.

A Taxi can accommodate a maximum of 4 passengers.

A Metro can accommodate a maximum of 500 passengers.

Fare calculation rules are different for different vehicle types.

Revenue should be updated whenever a booking is completed.
-----------------------------------------------------------------------------------------------
The system should identify and display:

Vehicle with highest revenue.

Vehicle with maximum occupancy.

Vehicles currently under maintenance.
                              
Exception Scenarios
---------------------------------------------------------------------------------------------------- 
The application must handle appropriate exceptions for situations such as:
 
Booking a passenger in a full vehicle.

Entering invalid travel distance.

Searching for a non-existing pas
enger.

Searching for a non-existing vehicle.

Performing an unsupported operation on a vehicle.

Invalid menu selection.
